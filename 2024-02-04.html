해시란?

해시는 데이터를 압축하여 고정된 크기의 출력 값을 생성하는 함수입니다.

  • 데이터의 무결성을 확인
  • 암호학적 보안 제공

해시의 특징

  • 단방향성: 해시값에서 원래 데이터를 되돌릴 수 없음
  • 충돌 저항성: 서로 다른 입력 값이 같은 해시 값을 가질 확률이 극히 낮음
  • 빠른 연산 속도: 블록체인에서 트랜잭션 검증과 무결성 확인에 사용됨

해시의 역할

  • 블록의 고유 식별자로 활용
  • 트랜잭션 무결성 검증
  • 머클 트리 구조에서 데이터 검증

블록체인에서 사용되는 대표적인 해시 함수

  • SHA-256: 비트코인, 이더리움 등에서 사용
  • Keccak-256 (SHA-3): 이더리움에서 사용
  • RIPEMD-160: 비트코인 주소 생성 과정에서 사용

작업 증명 (PoW)

블록체인의 합의 알고리즘 중 하나로, 새로운 블록을 추가할 때 연산 작업(채굴)을 수행해야 하는 방식입니다.

트랜잭션 ID (TXID)

블록체인에서 특정 트랜잭션을 식별하는 고유한 해시 값입니다.

머클 트리 (Merkle Tree)

  • 트랜잭션 데이터를 효율적으로 검증하고 저장하는 트리 구조
  • 머클 루트를 생성하여 데이터 무결성 검증 가능

비트코인 주소 생성 및 디지털 서명

비트코인의 주소는 비대칭키 암호화를 기반으로 생성되며, 트랜잭션 무결성을 보장하는 역할을 합니다.

암호화 (Encryption)

데이터를 보호하기 위해 특정 알고리즘을 사용하여 변환하는 과정입니다.

  • 대칭키 암호화: 암호화와 복호화에 동일한 키 사용
  • 비대칭키 암호화: 공개키(Public Key)와 개인키(Private Key) 사용
