📌 비트코인의 구조

비트코인은 블록 단위로 데이터를 저장하며, 블록은 크게 블록 헤더(Header)와 트랜잭션 리스트(Transaction List)로 구성됩니다.

📌 블록 헤더 (Header) - 블록의 메타데이터

블록 헤더는 80바이트로 고정되며, 다음과 같은 요소로 구성됩니다.

  • 이전 블록 해시 (Previous Block Hash): 이전 블록의 해시 값, 블록 간 연결성을 제공하여 무결성을 보장
  • 머클 루트 (Merkle Root): 블록 내 모든 트랜잭션을 하나의 해시로 표현, 트랜잭션 무결성을 증명
  • 타임스탬프 (Timestamp): 블록이 생성된 시간, UNIX 타임스탬프 형식으로 저장
  • 난이도 타겟 (Difficulty Target): 현재 블록 생성 난이도, PoW(Proof of Work) 목표 정의
  • 논스 (Nonce): 채굴자가 변경하는 값, PoW 연산 수행
  • 버전 (Version): 블록이 사용하는 비트코인 프로토콜 버전 정보

📌 블록 바디 (Body) - 1MB~4MB

블록 바디에는 블록에 포함된 트랜잭션 데이터가 저장됩니다.

  • 코인베이스 트랜잭션 (Coinbase Transaction): 블록에서 첫 번째 트랜잭션, 채굴자에게 블록 보상(채굴 보상 + 트랜잭션 수수료) 지급
  • 일반 트랜잭션 (Transactions): 블록에 포함된 모든 비트코인 전송 기록

📌 트랜잭션 구조

비트코인은 입력(Input), 출력(Output), 수수료로 구성되며, 디지털 서명을 포함하여 보안성을 유지합니다.

📌 트랜잭션 주요 구성 요소

🔹 입력 (Input)

  • 트랜잭션 해시 (Transaction Hash): 이전 트랜잭션의 해시 값 (32바이트)
  • 출력 인덱스 (Output Index): 해당 트랜잭션이 참조하는 특정 출력
  • 스크립트 서명 (ScriptSig): 이전 트랜잭션 출력의 소유권을 증명하는 서명 및 공개키 포함

🔹 출력 (Output)

  • 출력 금액 (Value): 수신자가 받는 비트코인 금액 (단위: 사토시)
  • 잠금 스크립트 (ScriptPubKey): 수신자의 주소를 포함하여 비트코인을 잠그는 스크립트

📌 UTXO (Unspent Transaction Output)

비트코인은 계좌 기반이 아닌 UTXO 모델을 사용하여 코인의 잔액을 추적합니다.

🔹 UTXO의 주요 특징

  • 이중 지불 방지: 한 번 사용된 UTXO는 재사용 불가
  • 잔돈 개념 필요: 전송 후 남은 금액은 새로운 UTXO로 변환
  • 데이터 투명성과 무결성 유지: 모든 UTXO는 공개적으로 조회 가능

📌 UTXO의 장점과 단점

🔹 장점

  • 이중 지불 방지 → 각 UTXO는 한 번만 사용 가능하여 보안성이 높음
  • 투명성과 보안성 → 모든 트랜잭션이 블록체인에 기록됨
  • 병렬 처리 가능 → 독립적인 UTXO 구조 덕분에 병렬 트랜잭션 처리 가능

🔹 단점

  • 복잡한 잔돈 관리 → 트랜잭션 처리 시 추가적인 데이터 필요
  • 저장소 부담 → 전체 네트워크의 UTXO 데이터를 유지하는 데 많은 저장소 필요

📌 머클 트리 (Merkle Tree)

트랜잭션 데이터를 계층적으로 해시하여 저장하는 트리 구조입니다.

  • 머클 루트 (Merkle Root): 블록 내 모든 트랜잭션을 대표하는 최상단 해시 값
  • 데이터 무결성 보장: 머클 루트 값이 변하면 트랜잭션이 변경되었음을 의미
  • 빠른 데이터 검증 (머클 프루프, Merkle Proof): 전체 블록 다운로드 없이 특정 트랜잭션 포함 여부 검증
  • 트랜잭션 데이터 저장 효율성 증가: 개별 트랜잭션 해시만 저장하면 되므로 데이터 저장 공간 절약
  • 블록체인 구조 최적화: 모든 트랜잭션을 한 번에 처리하는 것이 아니라 해시값을 활용하여 연산량 감소

📌 머클 트리 생성 과정

머클 트리는 단계적으로 해시를 생성하여 최종 머클 루트를 형성하는 구조입니다.

  • 각 트랜잭션 데이터를 해시(SHA-256)하여 리프 노드(Leaf Node) 생성
  • 두 개의 해시 값을 결합하여 다시 해시 → 부모 노드 생성
  • 이 과정을 반복하여 최종적으로 하나의 해시 값(머클 루트)이 남을 때까지 계속 수행

📌 머클 루트 (Merkle Root)

머클 트리의 최상단(루트) 해시 값으로, 블록 내 모든 트랜잭션을 대표하는 역할을 합니다.

  • 블록 내 모든 트랜잭션의 무결성을 증명
  • 머클 루트 값이 변하면 해당 블록의 트랜잭션이 변경되었음을 의미
  • 블록체인의 조작이 불가능하도록 보안 강화

📌 머클 트리의 주요 기능

  • 데이터 무결성 보장: 블록 내 트랜잭션이 변경되지 않았음을 보장
  • 빠른 데이터 검증 (머클 프루프, Merkle Proof): 전체 블록을 다운로드하지 않고도 특정 트랜잭션이 포함되었는지 검증 가능
  • 트랜잭션 데이터 저장 효율성 증가: 전체 트랜잭션을 저장하지 않고도 해시값만 저장 가능
  • 블록체인 구조 최적화: 해시 값을 사용하여 연산량을 줄이고 보안성을 유지
