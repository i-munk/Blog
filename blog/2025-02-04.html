📌 블록체인의 기본 구조

해시(Hash)는 데이터를 압축하고 고정된 크기의 출력 값을 생성하는 방식으로, 데이터의 무결성을 확인하고 암호학적 보안을 제공합니다.

📌 해시의 주요 특징

  • 단방향성: 해시값에서 원래 데이터를 되돌릴 수 없음
  • 충돌 저항성: 서로 다른 입력 값이 같은 해시 값을 가질 확률이 극히 낮음
  • 빠른 연산 속도: 블록체인에서 해시는 빠르게 계산되어 트랜잭션 검증과 무결성 확인에 사용

📌 해시의 역할

  • 블록의 고유 식별자 역할: 블록체인의 각 블록은 이전 블록의 해시 값을 포함하여 연결
  • 트랜잭션 무결성 검증: 트랜잭션이 해시 값을 사용하여 기록됨
  • 머클 트리 구조: 트랜잭션들의 해시 값을 결합하여 최종적으로 하나의 해시 값(머클 루트) 생성

📌 블록체인 대표 해시 함수

  • SHA-256: 비트코인, 이더리움 등에서 사용, 256비트의 고정된 길이 출력 값을 생성
  • Keccak-256 (SHA-3): 이더리움에서 사용되며, SHA-256과 구조적으로 차이가 있음
  • RIPEMD-160: 비트코인의 주소 생성 과정에서 사용됨 (SHA-256 후 RIPEMD-160 적용)

📌 블록 해시 (Block Hash)

해당 블록의 고유한 식별자로 사용되는 해시값이며, 아래 데이터를 결합하여 생성됩니다:

  • 이전 블록의 해시값
  • 머클 루트 (Merkle Root): 블록 내 모든 트랜잭션의 요약 값
  • 타임스탬프 (Timestamp)
  • 난이도 목표 (Difficulty Target)
  • 논스 (Nonce)

📌 작업 증명 (Proof of Work, PoW)

블록체인의 합의 알고리즘 중 하나로, 새로운 블록을 추가할 때 특정한 연산 작업(채굴)을 수행해야 하는 방식입니다. 이를 통해 네트워크의 보안을 유지하고, 트랜잭션을 검증하며, 분산 합의를 이룰 수 있습니다.

📌 트랜잭션 ID (TXID)

블록체인에서 특정 트랜잭션을 식별하는 고유한 해시 값으로, 입력, 출력, 금액 등의 데이터가 포함됩니다.

📌 머클 트리 (Merkle Tree)

블록체인에서 트랜잭션 데이터를 효율적으로 검증하고 저장하기 위해 사용되는 트리 구조입니다.

  • 각 노드는 트랜잭션 데이터의 해시 값
  • 부모 노드는 자식 노드 두 개의 해시 값을 결합하여 다시 해싱

📌 비트코인 주소 생성

비트코인의 주소 체계는 비대칭키 암호화를 기반으로 생성되며, 공개키와 비밀키를 사용합니다.

📌 디지털 서명

트랜잭션의 무결성을 보장하고 소유권을 증명하는 암호화 기술입니다.

🔹 디지털 서명의 동작 과정

  • 해싱(Hashing): 원본 메시지를 해시 함수로 변환하여 해시 값 생성
  • 서명(Signing): 생성된 해시 값을 서명자의 개인키로 암호화하여 디지털 서명 생성
  • 검증 (Verification): 수신자는 서명자의 공개키를 사용하여 서명이 올바른지 검증

📌 암호화 (Encryption)

데이터를 보호하기 위해 원본 정보를 특정 알고리즘을 사용하여 변환하는 과정입니다. 암호화된 데이터는 허가된 사용자만 복호화(Decryption)하여 읽을 수 있습니다.

📌 암호화의 유형

🔹 대칭키 암호화

암호화와 복호화에 동일한 키를 사용하며, 빠르고 효율적이지만 키 공유가 어려운 단점이 있습니다.

🔹 비대칭키 암호화

공개키(Public Key)와 개인키(Private Key)를 사용하며, 공개키로 암호화하고 개인키로 복호화합니다.
