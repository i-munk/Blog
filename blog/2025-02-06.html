📌 블록체인의 작동원리

블록체인은 데이터를 중앙집중형 원장 방식이 아닌 분산 원장(Distributed Ledger) 방식으로 관리하는 시스템입니다.

📌 중앙 집중형 원장

중앙 기관(은행, 회사, 정부 등)이 데이터를 관리하는 방식입니다.

🔹 특징

  • 중앙화된 신뢰: 중앙 기관이 거래 데이터를 관리 및 보증
  • 단일 장애점 (Single point of failure): 중앙 서버가 장애를 겪으면 데이터 유실 위험
  • 효율성: 중앙 기관이 데이터를 직접 관리하여 빠른 데이터 처리
  • 투명성 부족: 참여자가 데이터를 직접 검증하거나 확인할 수 없음

📌 분산 원장 (Distributed Ledger)

중앙 기관 없이 네트워크 참여자들이 데이터를 분산 저장하고, 동기화하며, 관리하는 방식입니다.

🔹 특징

  • 분산성: 네트워크의 여러 노드에 동일한 데이터 저장
  • 변경 불가능성: 기록된 데이터 변경 및 삭제가 어려움 (무결성 보장)
  • 투명성: 모든 참여자가 동일한 데이터를 볼 수 있어 신뢰 제공
  • 탈중앙화: 중앙 기관 없이 합의 메커니즘을 통해 데이터 관리

📌 분산 원장의 종류

  • 퍼블릭 분산원장: 누구나 참여 가능 (예: 비트코인, 이더리움)
  • 프라이빗 분산원장: 제한된 참여자만 접근 가능 (예: 하이퍼레저)
  • 컨소시엄 분산원장: 특정 그룹(기업 연합 등) 내에서 공유 (예: 기업 블록체인 네트워크)

📌 분산 원장의 활용 사례

  • 금융: 비트코인, 스마트 컨트랙트, 중앙은행 디지털화폐(CBDC)
  • 공급망 관리: 제품의 생산부터 유통까지 투명하게 추적
  • 헬스케어: 의료 기록을 안전하게 관리하고 공유
  • 전자 투표: 조작이 불가능한 온라인 투표 시스템 구축
  • 저작권 보호: 음악, 영상, NFT 등의 디지털 자산 보호

📌 노드 (Node) - 블록체인 네트워크 참여자

노드는 데이터를 저장하고, 관리하며, 다른 노드와 데이터를 교환하는 역할을 합니다.

🔹 노드의 역할

  • 트랜잭션 검증: 새로운 트랜잭션이 유효한지 확인
  • 블록 저장: 블록체인 데이터를 저장
  • 네트워크 전파: 트랜잭션과 블록을 다른 노드에 전송
  • 합의 프로세스 참여: PoW, PoS 같은 합의 알고리즘 수행

📌 노드의 유형

  • 풀 노드 (Full Node): 블록체인의 모든 데이터를 저장하고 검증
  • 라이트 노드 (Light Node, SPV): 블록 헤더만 저장하여 빠른 검증 수행
  • 마이닝 노드 (Mining Node): PoW를 통해 블록을 생성
  • 검증 노드 (Validator Node): PoS 네트워크에서 블록을 검증하고 합의 참여
  • 아카이브 노드 (Archive Node): 모든 블록체인 기록과 상태 데이터를 저장

📌 블록 생성 과정 & 합의 알고리즘

🔹 비잔틴 장애 허용 (BFT, Byzantine Fault Tolerance)

네트워크 내 일부 노드가 악의적으로 행동해도 올바른 합의에 도달할 수 있도록 보장

🔹 PoW (작업 증명, Proof of Work)

  • 채굴자가 연산을 수행하여 블록을 추가하는 방식
  • 높은 보안성을 유지하지만 연산 비용이 큼
  • 예: 비트코인, 이더리움 1.0

🔹 PoS (지분 증명, Proof of Stake)

  • 보유한 코인(지분)에 따라 블록을 검증하는 방식
  • 에너지 소비가 적고 속도가 빠름
  • 예: 이더리움 2.0, 솔라나

📌 비잔틴 장애 허용 (BFT, Byzantine Fault Tolerance)

분산 시스템에서 일부 노드가 악의적으로 행동해도 나머지 정직한 노드들이 올바른 합의에 도달할 수 있도록 보장하는 메커니즘입니다.

  • 분산 원장 시스템에서는 일부 노드가 거짓 정보를 제공할 수 있음
  • BFT는 악의적인 노드가 존재해도 시스템이 정상적으로 작동하도록 보장
  • PoW와 PoS 모두 BFT를 해결하는 방식으로 동작

📌 PoW (작업 증명, Proof of Work)

블록체인 네트워크에서 거래를 검증하고 새로운 블록을 생성하는 합의 알고리즘입니다.

  • 채굴자들이 연산을 수행하여 블록을 추가하는 방식
  • 높은 보안성을 유지하지만 연산 비용이 큼
  • 비트코인, 이더리움 1.0 등이 사용
  • SHA-256 해시 퍼즐을 풀어야 블록 생성 가능
  • 51% 공격 위험이 있지만, 실현이 어려울 정도로 높은 연산력이 필요

📌 PoW 작동 방식

  • 사용자가 트랜잭션을 발생시키면 네트워크에 전파
  • 트랜잭션이 메모리 풀(Mempool)에 대기
  • 채굴자가 해시 퍼즐을 풀기 위해 연산 수행
  • 해시값이 특정 목표보다 작을 경우 블록 생성 성공
  • 새 블록이 네트워크에 전파되고 검증 후 체인에 추가
  • 채굴자는 블록 보상과 트랜잭션 수수료를 획득

📌 PoS (지분 증명, Proof of Stake)

PoW의 높은 연산 비용과 에너지 소비 문제를 해결하기 위해 등장한 합의 알고리즘입니다.

  • 보유한 코인(지분)에 따라 블록을 검증하는 방식
  • 에너지 소비가 적고 속도가 빠름
  • 블록 생성자는 랜덤으로 선택되며, 보유 지분이 많을수록 선정 확률 증가
  • 이더리움 2.0, 카르다노, 솔라나 등이 사용

📌 PoS 작동 방식

  • 네트워크 참여자가 일정량의 코인을 스테이킹(예치)하여 검증자로 등록
  • 검증자는 랜덤으로 선택되어 블록을 생성
  • 정확한 블록을 추가하면 보상을 받고, 부정 행위를 하면 스테이킹한 코인을 잃음
  • 에너지 소비가 적고, 보안성과 확장성이 뛰어남

📌 PoW vs PoS

기준 PoW PoS
합의 방식 작업 증명 (연산력 기반) 지분 증명 (코인 보유량 기반)
보상 방식 채굴자가 블록 생성 시 보상 획득 검증자가 블록 생성 시 보상 획득
에너지 소비 높음 (채굴에 많은 전력 필요) 낮음 (전력 소모 거의 없음)
51% 공격 이론적으로 가능하지만 비용이 매우 큼 경제적으로 실현 불가능
대표 예시 비트코인, 이더리움 1.0 이더리움 2.0, 솔라나

📌 PoS의 장점

  • 에너지 효율성 증가 → 채굴이 필요하지 않아 전력 소비가 낮음
  • 트랜잭션 처리 속도 증가 → 블록 생성 속도가 빠름
  • 공격 비용 증가 → 51% 공격이 어렵고 보안성이 강화됨

📌 PoS의 단점

  • 부자 우선 시스템 → 많은 코인을 보유한 참여자가 더 많은 블록을 생성
  • 새로운 검증자에게 불리함 → 처음 시작하는 사용자는 검증자로 선정되기 어려움
